Esta primera tarea consiste en familiarizarse un poco con herramientas que deben ser comunes para el desarrollo de cualquier proyecto. En las siguientes instrucciones posiblemente hayan algunos términos desconocidos si nunca has trabajado con Git; queda de tu parte investigar lo que significan para poder realizar la actividad.

Como parte práctica de esta tarea, tendrás que seguir los siguientes pasos:

  1. Crear una cuenta en github.
  2. Realizar un fork en github de este repo: http://github/devlabs-usb/tarea-1.
  3. Clonar tu propio fork usando git clone.
  4. Hacer merge del branch conflict-branch que existirá en tu repositorio a tu branch gh-pages (ver git help checkout y git help merge).
  5. Resolver los "merge conflicts" que surgirán al hacer el merge.
  6. En tu propio repositorio, rellena tu archivo "index.html" con las respuestas a las preguntas teóricas que se encuentran acá.
  7. Hacer push de los cambios a "index.html" a tu repositorio cuando termines de rellenar las respuestas.
  8. Enviarnos un correo a devlabs-usb@talpor.com con el URL de tu repositorio en GitHub, tu nombre y carnet. Bonus: Si entiendes como funcionan los git pages, mándanos el link a tu página de respuestas.

3. En un contexto de la herramienta de control de versiones Git: definir 'working directory', 'index', 'local repo', 'remote repo', 'dirty working tree' y 'merge conflict'.

Working Directory: es uno de los tres componentes de un repositorio local, en él se almacenan los archivos de un proyecto cuando el usuario realiza un checkout.

Index: también conocido como cache o staging area . Es uno de los tres componentes de un repositorio local. Concretamente, es un archivo que almacena información relacionada a lo que irá en el próximo commit, es decir, es donde están los cambios realizados a un conjunto de archivos antes de que sean llevados al repositorio (con git commit).

Local Repo: es el repositorio que se encuentra en la computadora del usuario. Este contiene las diversas versiones de un proyecto

Remote Repo: es el repositorio que tiene diversas versiones de un proyecto alojadas en algún sitio de internet (o red local). Además de que se pueden tener varios de ellos, es posible establecer permisos (de sólo lectura o de lectura y escritura). Generalmente son utilizados para la colaboración en los proyectos.

Dirty Working Tree: 'The Working Tree' es el conjunto de archivos con los que se esta trabajando. Ahora bien, cuando se realiza checkout de un branch, estos archivos son reemplazados por los contenidos en el branch. Entonces, 'Dirty Working Tree' es la situación en la que el checkout a un branch particular reemplazará los archivos en 'The Working Tree' ocasionando, posiblemente, la perdida de los cambios realizados en ellos.

Merge Conflict: es un error anunciado por Git cuando no es capaz de determinar, dada dos versiones de un archivo, con cuál debe quedarse. Es entonces, cuando se necesita la intervención del usuario para resolver el conflicto.
